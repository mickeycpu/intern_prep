# 第2周学习总结：Python进阶与CLI实战

## 本周目标
掌握Python函数高级特性、面向对象编程（OOP）和异常处理，并运用所学开发一个功能完整的命令行记账本工具。

## 核心知识点
1.  **函数高级**：装饰器（`log_operation`）、生成器（`yield`）、`lambda`、`map`、`filter`、`sorted`。
2.  **面向对象**：类与对象、封装（私有属性`__`）、继承与多态、`super()`函数、方法重写。
3.  **异常处理**：`try-except-else-finally`结构、特定异常捕获（`ValueError`）、异常信息传递。
4.  **综合项目**：`AccountBook`类设计，实现收入/支出记录、余额查询、分类统计、数据持久化（内存）。

## 项目实战：个人记账本CLI
- **GitHub**: [链接到你的`day6_account_book_cli.py`]
- **功能**：添加收入/支出、查询余额、查看流水、分类统计。
- **关键设计**：
    - 使用装饰器自动记录操作日志。
    - 记录以`(时间, 类型, 金额, 分类)`元组存储，便于处理。
    - 所有用户输入和金额操作均有异常保护。
    - 使用列表推导式进行数据筛选和统计。

## 踩坑与解决
1.  **逻辑错误**：支出金额未存为负数、菜单选项重复。**解决**：明确数据模型，在方法内统一处理正负号。
2.  **数据格式**：记录字段索引变更导致统计错误。**解决**：定义清晰的元组结构，并在代码中同步更新所有引用点。
3.  **异常处理**：外层`try-except`吞掉具体错误。**解决**：将异常处理下沉到各个业务方法内部，主循环只处理流程。
4.  **Git协作**：`push rejected`、网络超时。**解决**：`git pull --rebase`合并远程更改；使用稳定网络或代理。
